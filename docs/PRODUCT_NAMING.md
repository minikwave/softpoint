# 제품·브랜드 가칭 명칭 정리

**2026년 4월** 기준 레포·UI 실태를 코드에서 확인한 정리다. 법적 상표 확정 전 **가칭**으로 보면 된다.

## 한 줄 요약

| 구분 | 가칭 | 비고 |
|------|------|------|
| **소비자-facing 앱 이름(한글)** | **쌓음** | 랜딩·디앱 네비 브랜드 (`Landing.tsx`, `App.tsx`) |
| **영문·기술 슬러그** | **saum** | 루트 패키지 `paypoint-saum`, 앱 폴더 `saum-webapp`, `index.html` 제목 `saum — PayPoint` |
| **엔진·레이어 이름** | **PayPoint** | 크레딧 SSOT·API·문서에서 제품 스택 이름 |
| **모음** | 스토리/카피 용어 | README 등 **「머지(통합)와 모음(쌓음)」**에서 “한 앱에 모아 쓴다”는 **제품 관점 설명**에 사용. **별도 영문 코드명 `moum`은 없음** |

즉, **가칭 제품(앱) 이름은 「쌓음」+ 기술 식별자 `saum`** 이고, **「모음」은 같은 방향을 말할 때 쓰는 말(모으다)**에 가깝지, 현재 레포의 공식 패키지명이 **moum**인 것은 아니다.

## 혼동 방지

- **머지 / 모음**: 문서에서 **통합 포인트(머지)** vs **한 앱에 쌓아 쓰기(모음)** 를 대비할 때 쓰는 **기획 언어**다.
- **쌓음**: 사용자가 실제로 보는 **앱 이름**에 가깝다.
- 향후 **「모음」을 앱 이름으로 통일**하거나 **`moum`으로 리네이밍**할 경우: `saum-webapp`·`paypoint-saum`·문서 전수·배포 URL을 함께 갱신해야 한다.

## 참고 위치

- UI: `apps/saum-webapp/src/pages/Landing.tsx`, `apps/saum-webapp/src/App.tsx`
- 패키지: 루트 `package.json`, `apps/saum-webapp/package.json`
- 문서: `README.md` 제목 `PayPoint (saum)`, 「제품 관점: 머지(통합)와 모음(쌓음)」
