15. PayPoint v1 개발 목표
v1은 이렇게 정의하는 게 좋습니다.

포인트를 적립·사용·교환하고, 모든 흐름을 원장·영수증·정책·감사 로그로 추적하는 Programmable Credit MVP

즉, v1에서 반드시 보여줘야 하는 것은 5개입니다.

1. 포인트 잔액
2. 상품권/기프티콘/게임머니 교환
3. Receipt 기반 사용 내역
4. 운영자 승인/예외 처리
5. 정산/준비금 대시보드 기초
AI agent와 stablecoin은 v1에서 완전 구현하지 않아도 됩니다.
다만 인터페이스와 narrative는 반드시 심어둬야 합니다.

  if (!/^[0-9]+$/.test(amount)) {

30. Ledger 서비스 실제 구조
apps/paypoint-api/src/services/ledger.ts

import { prisma } from "../lib/prisma"

export async function appendLedgerEntry(params: {
  userId: string
  accountId: string
  entryType: string
  amount: string
  receiptId?: string
  sourceType?: string
  sourceId?: string
  idempotencyKey?: string
  metadata?: unknown
}) {
  return prisma.$transaction(async (tx) => {
    const account = await tx.paypointAccount.findUnique({
      where: { id: params.accountId },
    })

    if (!account) {
      throw new Error("account not found")
    }

    const before = BigInt(account.balance.toString())
    const delta = BigInt(params.amount)

    const isDebit = [
      "SPEND",
      "LOCK",
      "REDEEM",
      "EXPIRE",
      "SETTLEMENT_DEBIT",
    ].includes(params.entryType)

    const after = isDebit ? before - delta : before + delta

    if (after < 0n) {
      throw new Error("insufficient balance")
    }

    const entry = await tx.ledgerEntry.create({
      data: {
        accountId: params.accountId,
        userId: params.userId,
        assetType: "PAYPOINT",
        entryType: params.entryType,
        amount: params.amount,
        balanceBefore: before.toString(),
        balanceAfter: after.toString(),
        receiptId: params.receiptId,
        sourceType: params.sourceType,
        sourceId: params.sourceId,
        idempotencyKey: params.idempotencyKey,
        metadata: params.metadata as any,
      },
    })

    await tx.paypointAccount.update({
      where: { id: params.accountId },
      data: { balance: after.toString() },
    })

    return entry
  })
}
추후에는 account row lock을 더 엄밀하게 처리해야 하지만, MVP에서는 이 구조부터 시작해도 됩니다.

import { prisma } from "../lib/prisma"

export async function createReceipt(params: {
  userId: string
  merchantId?: string
  agentId?: string
  intentType: string
  amount: string
  metadata?: unknown
}) {
  const receipt = await prisma.receipt.create({
    data: {
      userId: params.userId,
      merchantId: params.merchantId,
      agentId: params.agentId,
      intentType: params.intentType,
      status: "CREATED",
      amount: params.amount,
      assetType: "PAYPOINT",
      metadata: params.metadata as any,
    },
  })

  await prisma.receiptEvent.create({
    data: {
      receiptId: receipt.id,
      type: "CREATED",
      payload: {
        intentType: params.intentType,
        amount: params.amount,
      },
    },
  })

  return receipt
}

export async function updateReceiptStatus(params: {
  receiptId: string
  status: string
  payload?: unknown
}) {
  const receipt = await prisma.receipt.update({
    where: { id: params.receiptId },
    data: { status: params.status },
  })

  await prisma.receiptEvent.create({
    data: {
      receiptId: params.receiptId,
      type: params.status,
      payload: params.payload as any,
    },
  })

  return receipt
}
32. 상품 교환 서비스
apps/paypoint-api/src/services/redemption.ts

import { prisma } from "../lib/prisma"
import { getOrCreateAccount } from "./account"
import { createReceipt, updateReceiptStatus } from "./receipt"
import { appendLedgerEntry } from "./ledger"

export async function redeemProduct(params: {
  userId: string
  productId: string
  idempotencyKey: string
}) {
  const product = await prisma.creditProduct.findUnique({
    where: { id: params.productId },
  })

  if (!product || product.status !== "ACTIVE") {
    throw new Error("product not available")
  }

  const account = await getOrCreateAccount(params.userId)

  const receipt = await createReceipt({
    userId: params.userId,
    intentType: "CREDIT_REDEMPTION",
    amount: product.pricePaypoint.toString(),
    metadata: {
      productId: product.id,
      productType: product.productType,
      provider: product.provider,
    },
  })

  await updateReceiptStatus({
    receiptId: receipt.id,
    status: "AUTHORIZED",
  })

  await appendLedgerEntry({
    userId: params.userId,
    accountId: account.id,
    entryType: "REDEEM",
    amount: product.pricePaypoint.toString(),
    receiptId: receipt.id,
    sourceType: "CreditProduct",
    sourceId: product.id,
    idempotencyKey: params.idempotencyKey,
  })

  const mockCode = `MOCK-${Date.now()}`

  const redemption = await prisma.creditRedemption.create({
    data: {
      userId: params.userId,
      productId: product.id,
      receiptId: receipt.id,
      status: "FULFILLED",
      providerRef: `mock_${receipt.id}`,
      codeEncrypted: mockCode,
      metadata: {
        mock: true,
      },
    },
  })

  await updateReceiptStatus({
    receiptId: receipt.id,
    status: "COMPLETED",
    payload: {
      redemptionId: redemption.id,
    },
  })

  return {
    receipt,
    redemption,
  }
}
실서비스에서는 codeEncrypted를 반드시 암호화해야 합니다. MVP mock에서는 그대로 두되, 필드명은 미리 codeEncrypted로 잡아두는 게 좋습니다.
